LED显示屏基础知识

yukiCCC
LED LED

发布于：Jun 1, 2026

一、概述

1. LED 显示屏是什么

LED 显示屏（LED Display）是由大量 LED 灯珠组成的平面显示设备，通过控制每个像素点的亮灭和颜色，显示文字、图片、视频等内容。常见于广告屏、舞台屏、会议屏、交通诱导屏、体育场馆大屏等场景。

一套完整的 LED 显示屏系统通常包括：

理解各部件职责，是排查花屏、闪屏、黑屏等问题的基础。

二、核心硬件组成

1. 模组（Module）

模组是 LED 屏的最小显示单元，由 PCB 板、LED 灯珠、驱动 IC、排线接口等组成。

常见参数：

参数 说明
点间距（Pixel Pitch） 如 P2.5、P3、P4、P10，数值越小像素越密
分辨率 如 128×64、256×128（单模组像素）
扫描方式 静态、1/4 扫、1/8 扫、1/16 扫等
驱动 IC 如 ICN2038、MBI5124、SM16259 等

多个模组拼装成一个箱体。

2. 箱体（Cabinet）

箱体是模组的结构载体，同时承担防护、散热、安装固定作用。

常见类型：

  • 简易箱体：室内常用，精度要求相对低
  • 压铸铝箱体：精度高、平整度好，租赁屏常用
  • 前维护箱体：可从正面打开维护
  • 后维护箱体：从背面维护，固定安装常见

箱体关键要素：

项目 说明
尺寸 如 500×500mm、640×480mm、960×960mm
防护等级 室内 IP30~IP43，户外 IP65 及以上
散热设计 铝型材、风扇、散热孔
电源仓 开关电源安装位置
走线槽 电源线、网线、排线整理

箱体问题会直接影响整屏平整度、拼接缝、散热和稳定性。

3. 开关电源

为模组和接收卡供电。常见输出 5V，户外屏功率更大，需按整屏功耗冗余配置。

注意：

  • 电源功率不足 → 亮度下降、闪屏、重启
  • 电源质量差 → 纹波大，导致干扰、花屏
  • 多电源分区供电，避免单点故障整屏黑屏

三、发送卡

1. 发送卡是什么

发送卡（Sending Card）位于控制端，负责把电脑或播放设备的图像信号，转换成接收卡能识别的网络数据，并通过网线发送出去。

常见形态：

  • 插在 PC PCIe 插槽（如 NovaStar 发送卡）
  • 独立发送盒
  • 集成在视频处理器内部

2. 主要功能

  • 接收视频源（HDMI / DVI / DP / SDI 等）
  • 图像缩放、裁剪、拼接
  • 将画面按箱体/接收卡区域划分
  • 通过网口输出数据（常见千兆网口，可级联）

3. 关键参数

参数 说明
带载像素 单张发送卡能带的最大像素点数
网口数量 决定可连接多少条网线到接收卡
分辨率支持 如 1920×1200、3840×2160
冗余备份 双发送卡热备份（高端项目）

4. 常见品牌

  • 诺瓦（NovaStar）
  • 卡莱特（Colorlight）
  • 灵星雨（Linsn）
  • 摩西尔（Mosaic）等

四、接收卡

1. 接收卡是什么

接收卡（Receiving Card）安装在箱体内，接收发送卡传来的数据，再驱动对应模组上的 LED 灯珠显示。

一张发送卡通过网线可带多张接收卡，接收卡之间可级联（菊花链）。

2. 主要功能

  • 接收并解析网络数据包
  • 做 Gamma 校正、亮度调节、色温校正
  • 驱动模组上的行/列扫描
  • 回传箱体监控信息（温度、电压、工作状态）

3. 关键参数

参数 说明
带载范围 单卡能带多少像素（如 256×256）
扫描支持 支持 1/4、1/8、1/16 扫等
HUB 接口 75 接口，连接模组排线
校正支持 亮度校正、色度校正

4. 接收卡与模组的对应关系

配置软件中需设置：

  • 接收卡型号
  • 模组分辨率、扫描方式、数据组数
  • 接收卡带载宽高
  • 走线方式（几行几列、从哪开始）

配置错误是最常见的花屏、错位原因之一。

五、控制系统

1. 控制系统组成

LED 控制系统通常包括：

部分 作用
播放端 PC、异步播放盒、手机 APP
控制软件 LED 屏配置与调试软件
发送设备 发送卡 / 视频处理器
接收设备 接收卡
配电系统 总闸、时序上电、防雷

2. 同步控制 vs 异步控制

类型 说明 适用
同步控制 电脑实时输出画面，发送卡同步传输 会议、舞台、监控大屏
异步控制 播放盒脱机播放 U 盘/内置节目 广告屏、门楣屏

3. 常用配置软件

  • NovaLCT（诺瓦）
  • LEDSetting（卡莱特）
  • LED Studio 等

软件主要功能：

  • 发送卡 / 接收卡参数配置
  • 连屏（箱体拓扑）设置
  • 亮度、色温、校正
  • 固件升级、参数固化
  • 故障诊断与监控

4. 连屏（拓扑）配置

连屏是指定义每张接收卡对应屏幕上的哪一块区域，以及网线连接顺序。

连屏顺序或宽高设置错误，会出现画面错位、分块、镜像等问题。

六、信号与布线

1. 网线

  • 常用 超五类 / 六类网线
  • 单条网线传输距离一般建议 ≤100 米（可配合交换机或光纤延长）
  • 长距离使用 光纤收发器 + 光纤线

2. 排线（HUB 线）

接收卡与模组之间通过排线连接，接触不良会导致局部缺色、闪行、黑块。

3. 电源线

  • 强电与弱电分开走线
  • 地线可靠接地，避免干扰
  • 大功率屏建议分区供电、分时上电

4. 视频线

HDMI / DP 等连接电脑与发送设备，线材质量差或过长可能导致信号不稳定、丢色、闪屏。

七、常见故障及原因

1. 整屏黑屏

可能原因 排查方向
总电源未开或跳闸 检查配电柜、空开
控制端无信号 检查电脑、HDMI 线、发送卡
发送卡异常 指示灯、驱动、软件是否识别
发送卡网口无输出 更换网线、网口
所有接收卡无数据 检查首卡连接、参数是否固化
软件连屏错误 重新配置并发送

2. 局部黑屏 / 缺块

可能原因 排查方向
某张接收卡故障 交换接收卡定位
接收卡网线松动 重插 RJ45
开关电源故障 测 5V 输出
模组排线脱落 检查 HUB 排线
模组本身损坏 替换模组

3. 花屏 / 乱码 / 错位

可能原因 排查方向
接收卡参数与模组不匹配 扫描方式、数据组、OE 极性
连屏顺序错误 重新设置拓扑
网线质量差或超长 换线、加中继
接收卡固件版本不兼容 升级或统一版本
干扰严重 检查接地、电源、强弱电分离

4. 闪屏 / 抖动

可能原因 排查方向
电源功率不足或老化 测负载电流，更换电源
电源纹波大 换优质开关电源
接触不良 排线、网线、电源插头
刷新率设置不当 调整软件刷新参数
视频源帧率不稳定 换播放源或线缆

5. 偏色 / 亮度不一致

可能原因 排查方向
灯珠老化不一致 模组批次差异
未做亮度/色度校正 执行校正
接收卡 Gamma 设置不同 统一参数并固化
模组驱动 IC 故障 更换模组

6. 某一行或某一列不亮

可能原因 排查方向
行管/列管损坏 模组级维修或更换
驱动 IC 坏 更换模组
排线某一路不通 换排线
扫描参数错误 核对 1/N 扫配置

7. 无法控制 / 软件找不到设备

可能原因 排查方向
USB / 网线未连接 检查通信链路
IP 冲突 修改发送卡 IP
防火墙拦截 关闭或放行
驱动未安装 重装发送卡驱动
设备被锁 输入密码解锁

八、安装与维护要点

1. 安装前

  • 确认屏体尺寸、分辨率、点间距
  • 计算整屏功耗，配置足够电源
  • 规划网线、电源线走线
  • 钢结构或吊装承重需达标

2. 安装中

  • 箱体之间调平，保证平整度
  • 模组方向一致，避免装反
  • 接收卡、电源固定牢靠
  • 做好标签：网口编号、箱体编号

3. 验收测试

  • 全白、全红、全绿、全蓝测试坏点
  • 灰阶、渐变测试色彩
  • 视频播放测试流畅度
  • 长时间老化测试稳定性

4. 日常维护

  • 定期清洁屏体表面（软布，避免划伤）
  • 检查风扇、散热孔
  • 户外屏检查防水胶条、防雷
  • 备份接收卡参数和连屏文件

九、其他基础知识

1. 点间距与观看距离

点间距 常见场景 参考观看距离
P1.2 ~ P2 高端会议、演播室 1.5m ~ 3m
P2.5 ~ P4 室内商显 3m ~ 8m
P5 ~ P10 户外大屏 8m ~ 30m
P10 以上 远距离广告 30m 以上

2. 亮度单位

  • 室内屏：约 800~1500 cd/m²
  • 户外屏：约 4000~8000 cd/m² 或更高

3. 刷新率

刷新率越高，相机拍摄时越不易出现摩尔纹、扫描线。舞台、直播场景对刷新率要求更高，常见 1920Hz、3840Hz 及以上。

4. 灰度与 bit 数

灰度等级越高，画面过渡越细腻。常见 13bit、14bit、16bit 灰度处理。

5. 防护等级

  • IP43：室内一般防护
  • IP65：户外防尘防水常用等级

6. 名词速查

名词 含义
像素 最小显示单元
带载 一张卡能驱动的像素数量
扫描 逐行点亮 LED 的方式
数据组 模组数据通道分组
固化 参数写入接收卡/发送卡保存
校正 消除亮度/色度差异
冗余 备份设备，提高可靠性

十、小结

LED 显示屏不是「一块大显示器」，而是一套由 模组、箱体、电源、接收卡、发送卡、控制系统 组成的工程化系统。

排查故障时建议按链路顺序：

掌握发送卡、接收卡、控制系统和箱体的关系，再对照常见故障表，大多数现场问题都能快速定位。后续若结合《网络通讯协议与抓包工具学习》，还可以从网线通讯、IP 配置、长距离传输等角度进一步理解 LED 控制系统。
